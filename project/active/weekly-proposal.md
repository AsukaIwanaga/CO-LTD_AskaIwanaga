# PROJECT: 週次提案報告（毎週月曜）

**起票**: 2026-03-31
**ステータス**: 稼働中
**担当**: 経営企画部 (Michael)
**スケジュール**: 毎週月曜 04:30

---

## 概要

経営企画部が `project/will/` に蓄積した提案・企画案を、毎週月曜日に Don へ報告する。
報告はメール + ブリーフィング内に組み込む形式。

## フロー

1. **日常（随時）**: 経営企画部が改善案・新機能案を `project/will/` に格納
   - ファイル名: `will-YYYY-MM-DD-タイトル.md`
   - 内容: 課題 / 提案内容 / 期待効果 / 工数見積
   - Don の承認不要で自由に起票してよい

2. **毎週月曜 04:30**: 提案報告スクリプトが実行
   - `project/will/` の未報告ファイルを収集
   - 一覧をメールで Don に送信
   - 報告済みフラグ（frontmatter に `reported: YYYY-MM-DD`）を付与

3. **Don の判断**:
   - 「やろう」→ `will/` から `active/` に移動、着手
   - 「保留」→ `will/` に留置
   - 「不要」→ `will/` から削除 or `archive/` に移動

## will/ ファイルのテンプレート

```markdown
# WILL: [タイトル]

**起票**: YYYY-MM-DD
**起票者**: 経営企画部
**reported**: （未報告）

## 課題
現状の問題点や改善の余地

## 提案
具体的にやること

## 期待効果
何が良くなるか

## 工数見積
おおよその作業規模（小/中/大）
```
